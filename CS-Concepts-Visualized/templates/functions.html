Functions

A function is a block of code that performs a specific task. Functions allow us to break our program into smaller, more manageable chunks of code that can be reused throughout the program. Functions are useful for improving code readability, reusability, and modularity and generally just actually doing things.

What are Functions?

We can think of a function sort of like a set of instructions but these set of instructions, are isolated from the main function of the whole programs. Specific functions, do specific tasks that don't necessarily return their value to the user, but instead to the main function of the program. An example below is of a function that takes the value of two numbers and returns their sum.

Example code of a Function


                #include <stdio.h>

                    // we declare our function outside the main program
                    int addnumbers(int a, int b);

                    int main()
                    {
                        int num1, num2, result;

                        // user inputs numbers
                        printf("Enter two numbers: ");
                        scanf("%d %d", &num1, &num2);

                        // calling the "addnumbers" function we declared earlier
                        result = addnumbers(num1, num2);

                        // displaying only the result to the user
                        printf("The sum of %d and %d is: %d\n", num1, num2, result);

                        return 0;
                    }

                    // defining what our "addnumbers" function actually does "underneath the hood"
                    int addnumbers(int a, int b)
                    {
                        return a + b;
                    }

Use cases and benefits of Functions

Functions allow you to break your code into smaller, manageable pieces. This makes your program easier to read, debug, and maintain and this also makes it so, that you ony have to edit something once, outside of the main program. This makes the code secure too!

Sometimes, writing the same code over and over again can become a chore, its good practice but still sort of repetitive. Functions make it so that once a function is written, it can be reused multiple times without rewriting the same code. This also reduces redundancy and the chance for potential errors (and spending hours debugging later).

As you would see in the recursion section, functions can call themselves, as it the concept of recursion. This is useful in solving problems like factorials, tree traversals, and more complex algorithms to simplify your code and make the recursive aspect more readable.

We know how tedious handling errors and user input can get in programs, functions help to keep your program well-organized. For example, mathematical operations, string manipulations, or input/output handling can each be in their own functions so as to reduce the risk of your function interfering with any other part of your overall program.

Abstraction is basically, reducing each part of our code to its most basic function and separating every logical block (separation of concerns).
