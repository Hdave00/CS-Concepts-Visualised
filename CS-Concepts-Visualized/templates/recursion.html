Recursion

In the realm of Computer Science, recursion a concept of calling a function on itself. It's a tricky concept to understand and even harder to implement. It can be both advantageous and disadvantageous, depending on the situation. It's very important to have a "base case" established first or even hard coded to stop the program that is implementing the recursive code, to keep it from running for an indefinite time that may cause crashes or running loops.

What is Recursion?

Recursion is essentially, a method of solving problems where a function calls itself directly or indirectly. Each recursive call works on a smaller instance of the same problem until it reaches a base case, which stops further recursive calls. Recursion or recursive code is often used for tasks that can be broken down into similar subtasks, such as searching, sorting, or traversing hierarchical data structures, or can even be used to make youe code look prettier and more readable as it is broken down into a specific order of operation.

Example of a code using recursion to calculate the factorial of a number


                #include <stdio.h>

                    // function to calculate factorial using recursion
                    int factorial(int n)
                    {
                        if (n == 0 || n == 1)
                        { // base case
                            return 1;
                        }
                        return n * factorial(n - 1); // recursive case
                    }

                    int main()
                    {
                        int num = 5;
                        printf("factorial of %d is %d\n", num, factorial(num));
                        return 0;
                    }

Use cases and disadvantages of Recursion

The above code example is one such example of using recursion for calculating factorials (n!) . As it has a base case, that stops the recursion (n == 0 or n == 1) and it has a recursive case in the form of a function that keeps calling itself with a small value of ( n - 1 ), until the base case is satisfied. So if num = 5 , the the output in this case would be 120, as that is the factorial of 5.

In the algorithms section we saw how algorithms can be used to sort data. Recursion can help build those algorithms like MergeSort, BubbleSort and Linear search.

Recursion or recursive code has a few advantages, mainly it helps to simplify code related to linked tree traversal or liner data structures. It also helps in providing a solution for solving puzzles for games. As breifly mentioned earlier it also provied a simple solution for divide and conquer algorithms.

Recursion is a tricky concept to understand, I still have trouble with recursive code (in the unlikely case I actually use recursion) as the necessity for a base case, is easliy overlooked as it is often common to forget to add a base case to the function. We are humans learning everyday and making mistakes is part of the process, but recursion makes it easier to make such mistakes! Technically recursion also uses more memory than usual due to the function call stack (and it calling itself repeatedly), along with the risk of stack overflow if the base case is not defined properly.
